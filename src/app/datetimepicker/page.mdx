# DateTimePicker

## Critérios WCAG e eMAG
<Properties fullWidth>
  <Property name="WCAG 1.3.1 / 3.3.2 / eMAG Diretriz 6" type="Rótulos e instruções claras">
    O componente deve ter rótulos e instruções visíveis para ajudar os usuários a entender como selecionar datas e horas. Use rótulos apropriados para campos de data e hora e inclua sugestões de formato.
  </Property>
  <Property name="WCAG 2.1.1 / eMAG Diretriz 5" type="Navegabilidade por teclado">
    Certifique-se de que o DateTimePicker seja totalmente navegável via teclado, permitindo aos usuários selecionar uma data e hora sem a necessidade do mouse.
  </Property>
  <Property name="WCAG 2.4.7 / eMAG Diretriz 5" type="Estado de foco visível">
    O foco deve ser claramente indicado ao navegar pelo componente, com destaque visual para o campo ativo.
  </Property>
  <Property name="WCAG 1.4.3 / eMAG Diretriz 7" type="Contraste de cores">
    O texto do componente, assim como os dias selecionados no calendário, devem ter contraste adequado com o fundo para serem legíveis.
  </Property>
  <Property name="WCAG 3.3.3 / eMAG Diretriz 6" type="Erro e validação">
    Caso uma data ou hora inválida seja inserida, o componente deve fornecer uma mensagem de erro clara com uma sugestão de correção.
  </Property>
</Properties>
