# Tag

## Critérios WCAG e eMAG
<Properties fullWidth>
  <Property name="WCAG 1.1.1 / eMAG Diretriz 3" type="Texto Alternativo e Rotulagem Clara">
    A Tag deve ter um texto descritivo que explique claramente seu propósito. Se a tag for interativa (por exemplo, se puder ser removida ou clicada), deve ser fornecido um `aria-label` ou rótulo correspondente que descreva sua ação.
  </Property>
  <Property name="WCAG 2.1.1 / eMAG Diretriz 5" type="Navegabilidade por Teclado">
    Caso a Tag seja interativa, ela deve ser acessível via teclado. O usuário deve poder navegar até a tag e, se aplicável, acionar sua função (ex.: remover) usando apenas o teclado.
  </Property>
  <Property name="WCAG 2.4.7 / eMAG Diretriz 5" type="Estado de Foco Visível">
    Se a tag for interativa, seu estado de foco deve ser claramente visível, indicando quando o usuário está interagindo com ela via teclado.
  </Property>
  <Property name="WCAG 1.4.3 / eMAG Diretriz 7" type="Contraste de Cores">
    O texto da tag, bem como o fundo e a borda, devem ter contraste suficiente em relação ao fundo da página para garantir legibilidade, especialmente para usuários com baixa visão. A relação mínima de contraste recomendada é 4.5:1.
  </Property>
  <Property name="WCAG 3.3.1 / eMAG Diretriz 6" type="Estado de Foco Visível">
   Se a tag incluir um campo editável ou interativo (como a remoção de uma tag), deve fornecer feedback visual claro ao usuário (por exemplo, indicando que a tag foi removida ou selecionada com sucesso).
  </Property>
</Properties>
