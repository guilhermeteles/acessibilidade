# Collapse

## Critérios WCAG e eMAG
<Properties fullWidth>
  <Property name="WCAG 1.1.1 / eMAG Diretriz 3" type="Alternativas de texto para conteúdo não textual">
    Verifique se o ícone de colapso/expansão tem um texto alternativo apropriado (ex.: através de `aria-label` ou `aria-expanded`) para que leitores de tela possam identificar a ação associada ao ícone.
      </Property>
  <Property name="WCAG 2.1.1 / eMAG Diretriz 5" type="Navegação por teclado">
    Garanta que o componente seja navegável via teclado. O botão de colapso/expansão deve ser acessível utilizando apenas o teclado, e o estado de foco deve ser visível quando o usuário estiver interagindo com o componente.
  </Property>
  <Property name="WCAG 2.4.7 / eMAG Diretriz 5" type="Estado de foco visível">
    Adicione estilos visíveis para indicar o foco nos elementos interativos (botões de colapso). Isso garante que usuários de teclado saibam qual parte do componente está ativa.
  </Property>
  <Property name="WCAG 1.4.3 / eMAG Diretriz 7" type="Uso de cores adequadas">
    O contraste entre o botão de colapso/expansão e o fundo deve atender à relação mínima de contraste (4.5:1). Isso também vale para o conteúdo expandido.
  </Property>
  <Property name="WCAG 3.3.2 / eMAG Diretriz 6" type="Rótulos para campos e descrições">
    Certifique-se de que o colapso tenha rótulos claros e descrições adequadas para explicar o que será expandido/colapsado.
  </Property>
</Properties>
